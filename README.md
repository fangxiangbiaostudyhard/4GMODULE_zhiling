# 4G 模块常用 AT 指令使用教程

本文档汇总了 4G 模块常用的 AT 指令，包含短信发送、中文短信发送、GPS 定位等操作示例，并提供 Python 脚本用于 UCS2 编码转换，方便在串口助手中直接使用。

---

## 1. 基本设置

### 波特率

```text
115200
```

## 2. 发送短信

### 2.1 发送普通短信

```text
AT+CMGF=1
OK

AT+CSCS="GSM"
OK

AT+CMGS="18543444164"
> SOS location: w:111.111;w:123.234:
```

* **结束发送**：发送十六进制 `1A`（代表 Ctrl+Z），确认发送成功。

### 2.2 设置短信服务中心号码

```text
AT+CSCA="+8613800451500",145
AT+CSCA?
+CSCA: "+8613800451500",145
```

## 3. 发送中文短信

1. 设置短信为文本模式：

```text
AT+CMGF=1
```

2. 设置字符集为 UCS2：

```text
AT+CSCS="UCS2"
```

3. 设置短信文本参数：

```text
AT+CSMP=17,167,0,8
```

4. 转换号码和内容为 UCS2：

```text
18543444164 → "00310038003500340033003400340034003100360034"
你好 → "4F60517B"
```

5. 发送指令：

```text
AT+CMGS="00310038003500340033003400340034003100360034"
```

6. 出现 `>` 提示后输入 UCS2 内容：

```text
4F60517B
```

7. 结束并发送：发送十六进制 `1A`（Ctrl+Z）。

### 3.1 Python 脚本自动生成 UCS2 指令

```python
def to_ucs2_hex(text):
    """
    将任何文本（中文、数字、英文）转换为大端序的 UCS2 十六进制字符串
    """
    return text.encode('utf-16-be').hex().upper()


def generate_at_commands(phone, message):
    """
    生成可用于串口助手的 AT 指令
    """
    ucs2_phone = to_ucs2_hex(phone)
    ucs2_msg = to_ucs2_hex(message)

    print("--- 转换结果 ---")
    print(f"原始号码: {phone}")
    print(f"UCS2号码: {ucs2_phone}")
    print(f"原始短信: {message}")
    print(f"UCS2内容: {ucs2_msg}")
    print("\n--- 串口助手操作步骤 ---")
    print(f"1. 输入指令: AT+CMGS=\"{ucs2_phone}\"")
    print(f"2. 出现 > 后输入内容: {ucs2_msg}")
    print(f"3. 最后发送十六进制: 1A (即 Ctrl+Z)")


# --- 测试使用 ---
if __name__ == "__main__":
    target_phone = "18746554636"
    target_message = "丁一丁"
    generate_at_commands(target_phone, target_message)
```

## 4. 获取 GPS 位置信息

### 4.1 基本操作

```text
AT
OK

AT+QGPS=1
OK
```

### 4.2 查询 GPS 状态

```text
AT+QGPS=?
+QGPS: (0-1)
OK
```

### 4.3 获取 GPS 位置

```text
AT+QGPSLOC=?
+QGPSLOC: <UTC>,<latitude>,<longitude>,<HDOP>,<altitude>,<fix>,<COG>,<spkm>,<spkn>,<date>,<nsat>
OK
```

### 4.4 示例位置获取

```text
AT+QGPSLOC=0
+QGPSLOC: 121540.00,4542.6697N,12636.5093E,4.59,227.6,3,,1.166,0.639,301225,05
OK

AT+QGPSLOC=1
+QGPSLOC: 121546.00,4542.654900,N,12636.548330,E,3.19,206.5,3,,0.506,0.277,301225,07
OK

AT+QGPSLOC=2
+QGPSLOC: 121558.00,45.71077,126.60948,3.56,200.1,3,,0.819,0.449,301225,08
OK
```

### 4.5 简化获取流程

```text
AT
AT+QGPS=1
AT+QGPSLOC=2
```

### 4.6 十进制度表示法

* **纬度**：

  * 正数 → 北纬（N）
  * 负数 → 南纬（S）
* **经度**：

  * 正数 → 东经（E）
  * 负数 → 西经（W）

## 5. 补充说明

* **波特率**需与模块默认一致，一般 115200。
* **UCS2 编码**用于发送中文、特殊字符，确保字符集和短信参数正确。
* **GPS 模块**可能需要初始化时间，请先 `AT+QGPS=1` 启动。
* 串口助手中发送 Ctrl+Z 时，部分软件需要切换为 **HEX 模式**。
* 以上示例均可直接复制到串口助手或 Python 脚本中使用。

## 7. 常见错误与排查

| 错误现象                      | 可能原因                       | 解决方法                                                 |
| ------------------------- | -------------------------- | ---------------------------------------------------- |
| AT 指令无响应                  | 串口没连接、波特率错误                | 检查串口连接，确认波特率为 115200；尝试重启模块                          |
| 发送短信后没有收到                 | 短信服务中心号码错误，或者网络信号弱         | 使用 `AT+CSCA?` 检查短信服务中心；确保信号良好                        |
| 中文短信显示乱码                  | 未设置 UCS2 字符集或 AT+CSMP 参数不对 | 执行 `AT+CSCS="UCS2"` 并确认 `AT+CSMP=17,167,0,8`         |
| 发送短信后出现 `+CMS ERROR: 321` | 手机号或短信内容格式错误               | 手机号需转为 UCS2；确保内容长度在单条限制内                             |
| Ctrl+Z 无法发送               | 串口助手未切换 HEX 模式，或者发送方式不对    | 切换到 HEX 模式发送 0x1A，或使用助手的“发送 Ctrl+Z”功能                |
| GPS 定位失败                  | 模块未启动 GPS，或室内信号差           | 执行 `AT+QGPS=1` 启动 GPS；尽量在室外开阔地带获取信号                  |
| `+QGPSLOC` 返回空值           | GPS 未有有效定位                 | 等待模块定位完成，或者多次执行 `AT+QGPSLOC=2` 获取最新坐标                |
| 时间格式或经纬度错误                | 返回格式未理解                    | 注意纬度 N/S，正数北纬负数南纬；经度 E/W，正数东经负数西经；可用 Python 脚本转换为十进制 |

### 7.1 使用小技巧

1. **发送多条中文短信**：一次内容过长会被拆分，多条发送时需控制内容长度（一般 70 个汉字以内）。
2. **调试 GPS**：先执行 `AT+QGPS=1`，等待 10–20 秒，再查询位置，保证模块已接收足够卫星信号。
3. **UCS2 自动生成**：使用上文 Python 脚本，避免手动转换编码错误。
4. **串口助手推荐**：支持 HEX 模式和 Ctrl+Z 发送功能的软件，例如 SecureCRT、SSCOM、Termite 等。
